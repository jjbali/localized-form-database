Backup, Restore & About

Backup & Restore

Database export/import creates a literal, full-fidelity copy of your database(s) — every field of every record (including formatted messages, custom fields, and the dictionary) is preserved exactly, not simplified into plain text. Use "Export Database" to save just the database selected above as a single .json file you can move to another device; use "Export All Databases" to back up everything at once. "Import Database(s)" reads such a file back in — it always creates brand-new database(s) alongside whatever is already on this device, so nothing existing is ever overwritten.

To publish one of your own records, check "Make this record public" while creating or editing it in the Editor tab; uncheck it and save again to remove it from this list.

Terminal

A little offline hacking-style minigame — explore a fake filesystem with ls, cd, and cat, find passwords hidden in files to unlock locked directories, and track down the flags. Clear every flag in a round and the mainframe regenerates a brand new, harder layout just for you — it never ends. Your round, total flags collected, and score are saved to your account. Everything is simulated locally; no real files or network are touched. Type help to see every command.

Terminal Game — Admin

This is a never-ending game: once a player clears every flag in their current round, the mainframe procedurally generates a new, harder layout just for them, forever. There's a shared round 1 — the starting world every new player sees — and it's no longer edited as raw JSON. Instead, build it live in the terminal above using admin-only commands: editfs to enter edit mode, then mkdir, touch, write, rm, flag, lockdir, unlockdir and motd to shape it, savefs to publish it as round 1 for every new player, and stopedit to return to your own game. Type help in the terminal while logged in as admin to see the full list.
